# Live Activity Wireframes

## Overview
iOS Live Activities allow the Study Timer to display real-time timer information on the Lock Screen and Dynamic Island. This provides quick access to timer status without opening the app.

---

## Element Annotations

### Dynamic Island Elements

| Element | View | Description |
|---------|------|-------------|
| Timer Icon | Compact/Minimal | SF Symbol `timer`, indicates active session |
| Pause Icon | Compact/Minimal | SF Symbol `pause.fill`, shown when paused |
| Time Display | Compact/Expanded | MM:SS format, updates every second |
| Session Title | Expanded | User-defined session name |
| Progress Ring | Expanded | Circular progress indicator |
| Pause Button | Expanded | Pauses the active timer |
| Resume Button | Expanded | Resumes paused timer |
| Stop Button | Expanded | Ends session and saves to history |

### Lock Screen Elements

| Element | Description |
|---------|-------------|
| App Icon | Study Timer app icon for identification |
| Session Title | Name of current study session |
| Status Badge | Shows PAUSED or COMPLETED state |
| Large Time | Prominent MM:SS remaining display |
| Progress Bar | Horizontal bar showing completion percentage |
| Control Buttons | Pause/Resume and Stop actions |
| Dismiss Button | Clears the Live Activity after completion |

## Design Notes

1. **Color Coding**:
   - Running: Green/primary accent color
   - Paused: Yellow/orange warning color
   - Completed: Green success color

2. **Typography**:
   - Use SF Pro Rounded for time display
   - Ensure readability at Lock Screen distance

3. **Animations**:
   - Subtle pulse on progress ring while running
   - Smooth transitions between states

4. **Button Sizing**:
   - Minimum 44pt touch targets
   - Adequate spacing between buttons

5. **Accessibility**:
   - VoiceOver labels for all elements
   - Sufficient contrast ratios

6. **Update Frequency**:
   - Update every second while running
   - Stop updates when paused (battery efficiency)

## Implementation Notes

### ActivityKit Configuration

```swift
// Content State
struct TimerActivityContent: ActivityContent {
    let sessionTitle: String
    let remainingSeconds: Int
    let totalSeconds: Int
    let isPaused: Bool
    let isCompleted: Bool
}
```

### Update Triggers
- Timer tick (every second while running)
- Pause/Resume actions
- Stop action
- Timer completion

### End Conditions
- User stops the timer
- Timer naturally completes
- App force quit (graceful dismissal)
