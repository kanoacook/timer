# Session Input Wireframe

## Overview
The Session Input component allows users to name their study session before starting the timer. It can be implemented as a modal, inline input, or bottom sheet.

## Validation Rules

| Rule | Behavior |
|------|----------|
| Empty name | Use default "Study Session" |
| Too long (>50 chars) | Truncate with ellipsis in Live Activity |
| Special characters | Allow all (emoji support) |
| Whitespace only | Treat as empty, use default |

---

## Element Annotations

| Element | Description | Interaction |
|---------|-------------|-------------|
| Title Input | Text field for session name | Tap to focus, keyboard appears |
| Clear Button (â¨‰) | Clears input text | Tap to clear |
| Recent Suggestions | Pills showing past session names | Tap to autofill |
| Duration Presets | Quick duration selection | Tap to select |
| Start Button | Begins timer with current settings | Disabled if validation fails |
| Done Button | Dismisses keyboard, confirms input | Tap to confirm |

---

## Design Notes

1. **Auto-focus**: Keyboard should appear automatically when input is active
2. **Suggestions**: Show last 4-6 unique session names
3. **Default Value**: Pre-fill with last used name or "Study Session"
4. **Haptic Feedback**: Light haptic when selecting suggestions
5. **Accessibility**: Full VoiceOver support for input and suggestions

---

## Recommended Implementation

**For MVP**: Use **Option A (Inline Input)** for simplicity
- Minimal navigation
- Faster to start a session
- Cleaner UI

**Post-MVP**: Consider **Option B (Bottom Sheet)** if users want:
- Duration selection before starting
- More session metadata
